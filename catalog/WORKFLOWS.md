# Workflows Catalog

## Private-to-Public Curation

1. Start from private source policies/skills/subagents.
2. Remove private identifiers and workspace names.
3. Publish only generic markdown artifacts.
4. Manually review for naming and secret leakage before push.

## Selective Adoption

1. Copy only the files you need.
2. Place runtime overlays in your own tool directories.
3. Adapt skills/subagents to your local conventions.
4. Keep custom/project-specific logic in private overlays.
